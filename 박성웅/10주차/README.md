## Chapter 10. 로그인 및 회원 가입
## 핵심 키워드
### Spring Security
- 스프링 애플리케이션의 보안을 담당하는 도구로, 로그인과 권한 관리를 쉽게 할 수 있도록 돕는 것
- 모든 웹 요청에 대해 누가 왔는지 확인하고, 권한이 맞는지 검사하는 역할

### 인증(Authentication)과 인가(Authorization)
- 인증은 “너 누구냐?”를 확인하는 과정으로, 예를 들어 아이디와 비밀번호를 검사하는 것
- 인가는 “너는 이걸 할 수 있어”를 확인하는 과정으로, 로그인한 사람이 관리자 권한이 있는지 체크

### 세션과 토큰
- 세션은 서버가 로그인 상태를 기억하는 공간이며, 사용자는 브라우저 쿠키에 세션 ID만 저장합니다.
- 토큰은 로그인 정보를 담은 작은 증명서로, 사용자가 요청할 때 마다 이 증명서를 보여주면 서버가 확인

### 액세스 토큰(Access Token)과 리프레시 토큰(Refresh Token)
- 액세스 토큰은 짧은 시간 동안 로그인 상태를 인증할 때 쓰는 증명서
- 리프레시 토큰은 액세스 토큰이 만료되었을 때, 새로 발급받을 수 있도록 도와주는 긴 시간 유효한 증명서
